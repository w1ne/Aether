# Aether Release v0.2.5 - 2026-03-02

## 🚀 Highlights
This release marks a major milestone in Aether's portability, restoring native **iOS support** via a linkable static library and upgrading the core UI to **egui 0.33**.

## ✨ What's New
- **Native iOS Static Library**: The build pipeline now produces `libaether_ui.a`, configured for native iOS integration.
- **WGPU-Native Rendering**: iOS now defaults to the `wgpu` (Metal) backend, resolving legacy OpenGL-on-iOS stability issues.
- **egui 0.33 Ecosystem**: Upgraded core UI stack (`egui`, `eframe`, `egui_plot`, `egui_dock`) for improved performance and modern API support.
- **Hardware-Free Compilation**: Workspace crates (`aether-core`, `aether-agent-api`) now support a `hardware`-free build mode, enabling simulation-only use on mobile devices.

## 🛠️ Bug Fixes
- **eframe iOS Fix**: Resolved upstream compilation error in `eframe 0.28` related to `ViewportId::ROOT`.
- **CI Artifacts**: Fixed missing library output in release workflow.
- **CI Permissions**: Resolved GitHub Actions 403 errors by adding explicit `contents: write` permissions.
- **C-Dependency Resolution**: Migrated `syntect` to pure-Rust `fancy-regex` to avoid platform-specific C toolchain conflicts.

## 📱 Platform Specific Notes
### iOS
- Assets produced as a static library (`libaether_ui.a`) for static linking into Swift/ObjC projects.
- Uses `wgpu-only` rendering; OpenGL (`glow`) is disabled on mobile for stability.

### Desktop (Linux/macOS/Windows)
- Fully supports `glow` and `wgpu` backends.
- Includes full hardware probe support via `probe-rs` and `hidapi`.
